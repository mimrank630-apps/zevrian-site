# Wise Business KYC compliance sweep for pre-launch Zevrian site

This change converts a live-looking e-commerce site into an honest pre-launch brand presence for KYC review. It removes WhatsApp entirely, disables all Amazon purchase buttons with "Coming Soon" messaging, rewrites active-store language into future-tense, strips unverifiable product data (ratings, reviews, badges, comparison pricing), adds founder/LLC details, and updates structured data to reflect PreOrder availability. Twenty-three files touched across config, types, components, pages, and SEO.

**Watch for:** Present-tense fulfillment claims still live in `AmazonTrustSection` details text (confirmed). The launch badge pill uses `text-gold-600` on a semi-transparent gold background which may fail WCAG AA contrast (likely). The `ProductImage` component still renders a `badge` field from the type even though data was cleared — a dead code path that could resurface without compliance review (confirmed).

## High-level view

WhatsApp removal is complete — zero references survive in source.

Amazon button disablement is structurally sound across all call sites (navbar, hero, footer, trust section, product cards, product detail). The unused `href` prop remains in the `AmazonButton` interface, creating a false signal that the component can be re-linked.

Pre-launch language conversion covers the primary surfaces but two detail strings in `AmazonTrustSection` still speak in present tense about active order fulfillment ("Every order ships through Amazon's global logistics"), contradicting the "launches Q3 2026" framing everywhere else.

Product data cleanup removed all unverifiable fields from the 18-product catalog and switched schema.org to PreOrder. However, the rendering logic for `badge` and `compareAtPrice` remains in `ProductImage` and `ProductCard` — dormant now, but ready to silently resurface if product data is re-populated.

## Present-tense fulfillment claims in AmazonTrustSection

The heading and intro paragraph were correctly converted to future tense ("launching via Zevrian Direct", "We will use Amazon's world-class fulfillment"). But the two detail cards below still read:

```
"Every order ships through Amazon's global logistics network — the same reliability you already trust."
"Backed by Amazon's straightforward return policy and buyer protection on every purchase."
```

A Wise reviewer scanning for evidence of operations would flag these as implying current commercial activity. Fix: "Every order will ship through..." and "...on every purchase, once live."

**Confidence: confirmed** — lines visible in `src/components/sections/AmazonTrustSection.tsx`.

## Launch badge accessibility

The hero launch badge uses:

```tsx
<span className="... bg-gold/15 ... text-gold-600">
```

The `bg-gold/15` creates a semi-transparent pill over the dark hero (`charcoal-950`). Gold text on a gold-tinted background is a known low-contrast pattern for small text (14px requires 4.5:1). If the effective pill background resolves to approximately `#2a2418` and `gold-600` maps near `#9D8531`, the ratio hovers around 3.5:1.

**Confidence: likely** — cannot compute styles in sandbox, but the color combination follows a pattern that typically fails.

## Dead `badge` and `compareAtPrice` rendering paths

`ProductImage.tsx` still renders:

```tsx
{product.badge && (
  <span className="absolute left-4 top-4 ...">
    {product.badge}
  </span>
)}
```

`ProductCard.tsx` (line 48-50) and the detail page still render strikethrough pricing when `compareAtPrice` is present. Both fields are optional in the type and currently empty in data — but adding `badge: "Best Seller"` or `compareAtPrice: 59.99` to any product will silently create unverifiable claims with no compliance gate.

**Confidence: confirmed** — rendering logic present in `ProductImage.tsx` and `ProductCard.tsx`.

## Disabled button `aria-describedby` gap

The "Available Q3 2026" helper text below each disabled Amazon button is not programmatically associated with the button. Adding `aria-describedby` would let screen readers announce both the button label and the launch date context:

```tsx
<button aria-describedby="amazon-launch-date" ...>Coming Soon on Amazon</button>
<p id="amazon-launch-date">Available Q3 2026</p>
```

Minor enhancement — the button is already announced as disabled with its label.

**Confidence: confirmed** — no `aria-describedby` in `AmazonButton.tsx`.

## Residual ambiguous copy

Phrases that sit in the grey zone between planned-catalog marketing and implied active sales:

- "A selection of favorites from across the Zevrian catalog" — "favorites" implies customer preference data that doesn't exist.
- "Explore what we've built so far" (About CTA) — could read as "products you can buy now."
- "Pieces worth the counter space" — present-tense marketing for unreleased products.

None claim active transactions, but a conservative compliance posture would neutralize "favorites" at minimum.

**Confidence: possible** — unclear whether a Wise reviewer would flag marketing copy for planned products when no purchase path exists.
